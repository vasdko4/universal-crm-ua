# PowerFox — локальный запуск (демо-версия)

Интернет-магазин электроники (витрина + админ-панель) на Next.js 16, Postgres,
Drizzle ORM и Better Auth. Ниже — как поднять полную демо-копию магазина
PowerFox на локальном ПК: все 108 товаров с картинками, категории, заказы,
настройки, способы оплаты и доставки.

## Требования

- **Node.js 20+**
- **pnpm** (ставится автоматически через corepack, если отсутствует)
- **Docker** + **Docker Compose** — для локальной базы Postgres
  (либо свой Postgres, см. «Ручная настройка»)

## Быстрый старт на чистом сервере (одна команда) ⭐

```bash
bash scripts/setup.sh
# или:  pnpm setup
```

По умолчанию скрипт разворачивает **полную копию текущего магазина** — точный
снимок этого демо со всеми данными. Он сделает всё сам:

1. создаст `.env.local` и сгенерирует `BETTER_AUTH_SECRET`;
2. установит зависимости (`pnpm install`);
3. поднимет Postgres в Docker (`docker compose up -d db`);
4. **восстановит весь снимок БД из `db/dump.sql`** — схема + все данные:
   108 товаров с картинками, 24 категории, реальные заказы с историей,
   настройки магазина (оплата, доставка, уведомления, SEO), роли и
   учётные записи.

> Картинки товаров хранятся в Vercel Blob (публичные URL) и работают из
> локальной копии без дополнительной настройки.

После завершения запустите приложение:

```bash
pnpm dev
```

Откройте **http://localhost:3000** — магазин уже полностью заполнен и готов к
работе.

| Что            | Адрес                              |
| -------------- | ---------------------------------- |
| Витрина        | http://localhost:3000              |
| Админ-панель   | http://localhost:3000/admin        |

**Администратор:** `admin@powerfox.store` / `PowerFox2026!`

**Тестовый покупатель:** `vasadko3@gmail.com` / `12ew12ew`

> `db/dump.sql` — самодостаточный SQL-файл (схема + все данные + связи +
> индексы), сгенерированный из текущей базы. Чтобы пересоздать его из активной
> БД, выполните `pnpm db:dump`.

### Другие варианты установки

- **Только пустая схема + мастер настройки** (создать свой магазин с нуля):

  ```bash
  pnpm setup --schema     # или:  bash scripts/setup.sh --schema
  ```

  При первом заходе на **http://localhost:3000** откроется мастер установки
  (`/setup`): создаст администратора, задаст название магазина и ключ Нова Пошти.

- **Схема + встроенный демо-seed** (генерируемые демо-данные, не снимок):

  ```bash
  pnpm setup --seed       # или:  bash scripts/setup.sh --seed
  ```

- **Восстановить снимок в уже настроенную БД** (свой Postgres / OSPanel):

  ```bash
  pnpm db:restore         # DROP + восстановление из db/dump.sql в DATABASE_URL
  ```

## OSPanel и другой локальный Postgres (ввод данных БД в браузере)

Если вы используете **OSPanel** (модуль PostgreSQL) или любой уже установленный
Postgres, Docker не нужен. Данные подключения можно ввести прямо в браузере:

1. Установите зависимости и запустите приложение:

   ```bash
   pnpm install
   pnpm dev
   ```

   > Если файла `.env.local` ещё нет — создайте его из примера и задайте секрет:
   > `cp .env.example .env.local`, затем сгенерируйте `BETTER_AUTH_SECRET`
   > (`openssl rand -base64 32`). Поле `DATABASE_URL` можно оставить пустым.

2. Заранее создайте в OSPanel пустую базу (например, `techno_store`).

3. Откройте **http://localhost:3000**. Откроется шаг **«Подключение к базе
   данных»** — введите хост (`localhost`), порт (у OSPanel обычно `5432`), имя
   базы, пользователя и пароль. SSL для локальной базы оставьте выключенным.
   Мастер проверит подключение, создаст таблицы и сохранит `DATABASE_URL`
   в `.env.local`.

4. **Перезапустите** `pnpm dev` (Ctrl+C, затем снова `pnpm dev`) и нажмите
   «Проверить подключение» — откроется мастер создания администратора и
   настройки магазина.

## Ручная настройка (без Docker)

1. Скопируйте переменные окружения и заполните их:

   ```bash
   cp .env.example .env.local
   # укажите DATABASE_URL на свой Postgres и задайте BETTER_AUTH_SECRET:
   #   openssl rand -base64 32
   ```

2. Установите зависимости и примените схему:

   ```bash
   pnpm install
   pnpm db:setup        # только схема; магазин настраивается через мастер
   # или с демо-данными:
   pnpm db:seed
   ```

3. Запустите:

   ```bash
   pnpm dev
   ```

## Полезные команды

| Команда           | Действие                                                        |
| ----------------- | --------------------------------------------------------------- |
| `pnpm setup`      | Установка на чистый сервер: env + deps + БД в Docker + **полная копия текущей БД** (`db/dump.sql`) |
| `pnpm setup --schema` | То же, но только пустая схема + мастер настройки в браузере   |
| `pnpm setup --seed` | То же, но со встроенным демо-seed вместо снимка               |
| `pnpm db:setup`   | Применить только схему к базе из `DATABASE_URL`                  |
| `pnpm db:seed`    | Применить схему и демо-данные                                    |
| `pnpm db:reset`   | Полностью пересоздать схему (DROP + CREATE)                      |
| `pnpm db:restore` | Восстановить полный снимок текущей БД из `db/dump.sql` (DROP + restore) |
| `pnpm db:dump`    | Пересоздать `db/dump.sql` из активной базы `DATABASE_URL`        |
| `pnpm dev`        | Запустить приложение в режиме разработки                        |

## Структура данных

- `db/schema.sql` — все таблицы (товары, категории, заказы, отзывы, вопросы,
  способы доставки/оплаты, настройки магазина, таблицы Better Auth и т.д.).
- `db/seed.sql` — демо-данные: 6 товаров, 4 категории, способы доставки
  (Нова Пошта, Укрпошта) и оплаты, настройки магазина, тестовые заказы и админ.

## Важно про ключи и токены

Снимок `db/dump.sql` содержит настройки магазина **вместе с сохранёнными в
админке ключами** (SMTP-пароль почты, токены Telegram-бота и платёжных шлюзов,
ключ Нова Пошти). Это удобно для полноценного демо «как есть», но:

- не публикуйте репозиторий с этим дампом в открытом доступе;
- при передаче демо третьим лицам замените ключи в админке
  (Настройки → Уведомления / Платежи / Доставка) или очистите их перед
  пересозданием дампа.

## Мониторинг ошибок (Sentry)

Проект подключён к [Sentry](https://sentry.io) (`@sentry/nextjs`), но **выключен
по умолчанию** — без переменных окружения `SENTRY_DSN`/`NEXT_PUBLIC_SENTRY_DSN`
ничего никуда не отправляется, приложение работает как обычно.

Чтобы включить (рекомендуется на проде — иначе об ошибках оплаты/писем/webhook'ов
вы узнаете только от клиента):

1. Зарегистрируйтесь на sentry.io (бесплатного тарифа хватает для одного магазина).
2. Создайте проект с платформой Next.js, скопируйте DSN.
3. Впишите его в `SENTRY_DSN` и `NEXT_PUBLIC_SENTRY_DSN` в `.env.local`/`.env.production`.
4. Перезапустите приложение.

## Бэкап базы данных на проде

`scripts/vps-install.sh` сам настраивает ежедневный бэкап (systemd-таймер,
03:30 UTC) — `pg_dump` в `/var/backups/techno-store`, хранение 14 дней.
Это защищает только от порчи БД, но не от потери самого сервера/диска.

Чтобы бэкапы улетали ещё и на внешнее хранилище — настройте
[rclone](https://rclone.org) (`rclone config`, поддерживает S3, Backblaze B2,
Google Drive и т.д.) и задайте `RCLONE_REMOTE` в `.env.production`
(например `RCLONE_REMOTE="b2:my-bucket/techno-store-backups"`).

Восстановление из бэкапа:

```bash
pg_restore --clean --if-exists -d "$DATABASE_URL" /var/backups/techno-store/techno-store_<дата>.dump
```

## Заметки

- **Нова Пошта** работает «из коробки»: ключ API хранится в настройках доставки
  (в демо-данных). Свой ключ можно задать через `NOVA_POSHTA_API_KEY` в `.env.local`
  или в админке (Настройки → Доставка).
- **Письма**: если SMTP не настроен в админке, письма не отправляются, а логируются
  в консоль — для локальной разработки этого достаточно (в т.ч. коды сброса пароля).
- База данных Docker слушает порт **5433** на хосте, чтобы не конфликтовать с
  возможным локальным Postgres на 5432.

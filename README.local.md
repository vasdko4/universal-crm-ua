# PowerFox — локальный запуск (демо-версия)

Интернет-магазин электроники (витрина + админ-панель) на Next.js 16, Postgres,
Drizzle ORM и Better Auth. Ниже — как поднять магазин на локальном ПК:
чистая установка через мастер настройки или демо-наполнение фейковыми данными.

## Требования

- **Node.js 20+**
- **pnpm** (ставится автоматически через corepack, если отсутствует)
- **Docker** + **Docker Compose** — для локальной базы Postgres
  (либо свой Postgres, см. «Ручная настройка»)

## Быстрый старт на чистом сервере (одна команда) ⭐

```bash
bash scripts/setup.sh
# или:  pnpm setup
```

Скрипт сделает всё сам:

1. создаст `.env.local` и сгенерирует `BETTER_AUTH_SECRET`;
2. установит зависимости (`pnpm install`);
3. поднимет Postgres в Docker (`docker compose up -d db`);
4. применит схему БД (`db/schema.sql`).

После завершения запустите приложение:

```bash
pnpm dev
```

Откройте **http://localhost:3000** — на первом заходе автоматически откроется
**мастер установки**: название магазина, описание, ключ Нова Пошти, выбор
дизайна, SEO и админ-аккаунт (логин/пароль задаёте сами).

| Что            | Адрес                              |
| -------------- | ---------------------------------- |
| Витрина        | http://localhost:3000              |
| Админ-панель   | http://localhost:3000/admin        |

### Другие варианты установки

- **С демо-данными** (фейковые товары, категории, тестовые заказы):

  ```bash
  pnpm setup --seed
  # демо-админ: admin@techno.store / Admin12345 — сразу смените!
  ```

- **Перенос своего магазина**: создайте снимок на старом окружении
  (`pnpm db:dump` → `db/dump.sql`; файл в `.gitignore` и в репозиторий не
  попадает), перенесите его на сервер (scp) и выполните `pnpm db:restore`.

## Полезные команды

| Команда           | Действие                                                        |
| ----------------- | --------------------------------------------------------------- |
| `pnpm setup`      | Установка на чистый сервер: env + deps + БД в Docker + пустая схема (мастер настройки в браузере) |
| `pnpm setup --seed` | То же, но со встроенным демо-seed (фейковые данные)           |
| `pnpm db:setup`   | Применить только схему к базе из `DATABASE_URL`                  |
| `pnpm db:seed`    | Применить схему и демо-данные                                    |
| `pnpm db:reset`   | Полностью пересоздать схему (DROP + CREATE)                      |
| `pnpm db:restore` | Восстановить локальный снимок `db/dump.sql` (DROP + restore); файл не хранится в репозитории |
| `pnpm db:dump`    | Создать `db/dump.sql` из активной базы `DATABASE_URL` (для переноса; в `.gitignore`) |
| `pnpm dev`        | Запустить приложение в режиме разработки                        |

## Структура данных

- `db/schema.sql` — все таблицы (товары, категории, заказы, отзывы, вопросы,
  способы доставки/оплаты, настройки магазина, таблицы Better Auth и т.д.).
- `db/seed.sql` — демо-данные: 6 товаров, 4 категории, способы доставки
  (Нова Пошта, Укрпошта) и оплаты, настройки магазина, тестовые заказы и админ.

## Важно про ключи и токены

Снимок `db/dump.sql` (создаётся командой `pnpm db:dump`) содержит настройки
магазина **вместе с сохранёнными в админке ключами** (SMTP-пароль почты, токены
Telegram-бота и платёжных шлюзов, ключ Нова Пошти) и персональными данными
клиентов. Поэтому:

- файл находится в `.gitignore` и **никогда не коммитится в репозиторий**;
- переносите его только напрямую (scp/rsync) и удаляйте после восстановления;
- при передаче копии третьим лицам сначала очистите ключи в админке
  (Настройки → Уведомления / Платежи / Доставка).

## Мониторинг ошибок (Sentry)

Проект подключён к [Sentry](https://sentry.io) (`@sentry/nextjs`), но **выключен
по умолчанию** — без переменных окружения `SENTRY_DSN`/`NEXT_PUBLIC_SENTRY_DSN`
ничего никуда не отправляется, приложение работает как обычно.

Чтобы включить (рекомендуется на проде — иначе об ошибках оплаты/писем/webhook'ов
вы узнаете только от клиента):

1. Зарегистрируйтесь на sentry.io (бесплатного тарифа хватает для одного магазина).
2. Создайте проект с платформой Next.js, скопируйте DSN.
3. Впишите его в `SENTRY_DSN` и `NEXT_PUBLIC_SENTRY_DSN` в `.env.local`/`.env.production`.
4. Перезапустите приложение.

## Бэкап базы данных на проде

`scripts/vps-install.sh` сам настраивает ежедневный бэкап (systemd-таймер,
03:30 UTC) — `pg_dump` в `/var/backups/techno-store`, хранение 14 дней.
Это защищает только от порчи БД, но не от потери самого сервера/диска.

Чтобы бэкапы улетали ещё и на внешнее хранилище — настройте
[rclone](https://rclone.org) (`rclone config`, поддерживает S3, Backblaze B2,
Google Drive и т.д.) и задайте `RCLONE_REMOTE` в `.env.production`
(например `RCLONE_REMOTE="b2:my-bucket/techno-store-backups"`).

Восстановление из бэкапа:

```bash
pg_restore --clean --if-exists -d "$DATABASE_URL" /var/backups/techno-store/techno-store_<дата>.dump
```

## Заметки

- **Нова Пошта** работает «из коробки»: ключ API хранится в настройках доставки
  (в демо-данных). Свой ключ можно задать через `NOVA_POSHTA_API_KEY` в `.env.local`
  или в админке (Настройки → Доставка).
- **Письма**: если SMTP не настроен в админке, письма не отправляются, а логируются
  в консоль — для локальной разработки этого достаточно (в т.ч. коды сброса пароля).
- База данных Docker слушает порт **5433** на хосте, чтобы не конфликтовать с
  возможным локальным Postgres на 5432.

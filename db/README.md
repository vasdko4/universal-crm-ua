# База данных

В репозитории хранятся только «чистые» SQL-файлы без секретов и реальных данных:

| Файл         | Что это                                                        |
|--------------|----------------------------------------------------------------|
| `schema.sql` | Полная схема БД (41 таблица, ключи, индексы) — без данных      |
| `seed.sql`   | Демо-данные (фейковые товары, категории, заказы) для просмотра |
| `migrate.sql`| Инкрементальные миграции для существующих баз                  |

> **Важно:** `db/dump.sql` (полный снимок живого магазина) в репозиторий
> **не коммитится** — он в `.gitignore`, потому что содержит реальные ключи
> платёжных систем, пароли почты и персональные данные клиентов. Дамп — это
> локальный/серверный артефакт для переноса и бэкапа.

## Чистая установка (рекомендуется)

```bash
pnpm setup --schema     # env + deps + Postgres в Docker + пустая схема
pnpm dev
```

На первом заходе на http://localhost:3000 откроется мастер установки: там
создаётся магазин и админ-аккаунт.

## Демо-данные

```bash
pnpm setup --seed       # схема + фейковые товары/заказы
# демо-админ: admin@techno.store / Admin12345 (сразу смените!)
```

## Перенос своего магазина (dump/restore)

```bash
# на старом сервере / локально с доступом к живой базе:
pnpm db:dump            # создаст db/dump.sql из DATABASE_URL (.env.local)

# перенесите файл на новый сервер вручную (scp), затем:
pnpm db:restore         # DROP схемы + восстановление из db/dump.sql
```

Или вручную через psql:

```bash
createdb myshop
psql myshop -f db/dump.sql
```

Скрипт `scripts/dump-db.mjs` формирует значения средствами самого PostgreSQL,
поэтому все типы (jsonb, timestamptz, numeric, boolean) переносятся корректно.

## Настройка приложения

Создайте `.env.local` в корне проекта:

```bash
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/myshop"
BETTER_AUTH_SECRET="$(openssl rand -base64 32)"
```
